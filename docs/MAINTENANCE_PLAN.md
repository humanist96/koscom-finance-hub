## 코스콤 금융영업부 - 기술 현실 분석 및 바이브 코딩 기반 운영 방안

## 1. 현황 분석

### 1.1 시스템 개요

| 항목 | 내용 |
|------|------|
| 시스템명 | 금융동향파악 (Securities Intelligence Hub) |
| 목적 | PowerBase 고객사(증권사) 모니터링 및 영업 지원 |
| 배포 환경 | Vercel (서버리스) |
| 데이터베이스 | PostgreSQL (Neon 클라우드) |
| **서비스 유형** | **내부 직원용 서비스 (고객 대면 X)** |
| **안정성 요구수준** | **중간 (장애 시 업무 지연 수준)** |

### 1.2 서비스 방향성

```
┌─────────────────────────────────────────────────────────────┐
│  서비스 운영 방향                                             │
├─────────────────────────────────────────────────────────────┤
│  ✅ 내부 직원 전용 서비스 → 서비스 안정성 요구 수준 낮음        │
│  ✅ 고도화 완료 후 개발/개편 중지 → 운영 모드 전환              │
│  ✅ 최소 운영 비용으로 유지 가능한 구조 지향                    │
└─────────────────────────────────────────────────────────────┘

핵심 운영 목표:
1. 뉴스 수집: 완전 자동화 (크롤러 기반)
2. 계약 정보: 엑셀 업로드 방식으로 단순화 (수동 입력 최소화)
3. 인사 정보: 뉴스 기반 자동 추출 또는 수동 등록
```

### 1.3 기술 스택 복잡도 평가

| 기술 영역 | 사용 기술 | 난이도 | 바이브 코딩 적합도 |
|-----------|-----------|--------|------------------|
| 프론트엔드 | Next.js 16, React 19, TypeScript | ★★★ | 높음 |
| 백엔드 API | Next.js API Routes | ★★ | 높음 |
| 데이터베이스 | Prisma ORM + PostgreSQL | ★★★ | 중간 |
| AI 연동 | Anthropic Claude API | ★★ | 높음 |
| 웹 크롤링 | Puppeteer, Cheerio | ★★★★ | 낮음 |
| 인증 | NextAuth.js | ★★★ | 중간 |
| 배포/운영 | Vercel, Docker | ★★ | 높음 |

### 1.4 핵심 쟁점

```
┌─────────────────────────────────────────────────────────────┐
│  금융영업팀 현실                                              │
├─────────────────────────────────────────────────────────────┤

## 2. 바이브 코딩(Vibe Coding) 가능성 평가

### 2.1 바이브 코딩이란?

AI 코딩 어시스턴트(Claude, Copilot 등)를 활용하여 비개발자 또는 타 기술 개발자가 코드를 유지보수하는 방식

### 2.2 영역별 바이브 코딩 적합도

#### 바이브 코딩으로 충분히 가능한 영역 (70%)

| 작업 유형 | 예시 | 이유 |
|-----------|------|------|
| **UI 수정** | 텍스트 변경, 색상 조정, 레이아웃 수정 | Tailwind CSS 직관적, AI가 잘 이해 |
| **API 파라미터 수정** | 필터 조건 변경, 정렬 순서 변경 | Prisma 쿼리 명확함 |
| **증권사 데이터 추가** | 신규 증권사 등록, 정보 수정 | DB seed 파일 수정 |
| **환경 변수 변경** | API 키 교체, URL 변경 | .env 파일 수정 |
| **간단한 버그 수정** | null 체크, 조건문 수정 | AI가 에러 메시지 분석 |
| **알림 문구 수정** | 이메일/알림 텍스트 변경 | 문자열 수정 |
| **차트 설정 변경** | 차트 색상, 라벨 수정 | Recharts 설정 |

#### 바이브 코딩 가능하나 주의 필요 (20%)

| 작업 유형 | 위험 요소 | 권장 조치 |
|-----------|-----------|-----------|
| **새 API 엔드포인트 추가** | 보안, 인증 처리 | 코드 리뷰 필수 |
| **데이터베이스 스키마 변경** | 마이그레이션 실패 위험 | 스테이징 환경 테스트 |
| **인증 로직 수정** | 보안 취약점 | 보안 검토 필수 |
| **새 컴포넌트 생성** | 기존 패턴 불일치 | 기존 코드 참조 |

#### 바이브 코딩 부적합 - 전문가 필요 (10%)

| 작업 유형 | 이유 | 대안 |
|-----------|------|------|
| **크롤러 신규 개발** | 대상 사이트별 HTML 구조 분석 필요 | 외부 전문가 |
| **크롤러 고장 수리** | 웹사이트 구조 변경 대응 | 외부 전문가 |
| **성능 최적화** | 깊은 이해 필요 | 외부 전문가 |
| **보안 취약점 대응** | 전문 지식 필요 | 보안 전문가 |
| **대규모 리팩토링** | 아키텍처 이해 필요 | 외부 전문가 |

## 3. 극복 방안

### 3.0 고도화 완료 후 운영 전환 전략

```

│  2. 계약 정보 엑셀 업로드 기능 개발 ⭐ 신규                    │
│     - 표준 엑셀 템플릿 정의                                   │
│     - 엑셀 파일 업로드 → DB 자동 반영 기능                    │
│     - 데이터 검증 및 오류 리포트 기능                         │

│  3. 운영 편의 기능                                           │
│     - 관리자용 데이터 수동 입력/수정 UI                       │

✅ 뉴스 크롤링이 7일 이상 무장애 운영

## 5. 위험 관리

### 5.0 위험 수용 기준 (내부 서비스)

```
┌─────────────────────────────────────────────────────────────┐
│  내부 직원 서비스 → 위험 수용 수준 높음                        │
├─────────────────────────────────────────────────────────────┤
│  ✅ 1-2일 서비스 중단 허용 (외부 고객 없음)                   │
│  ✅ 데이터 갱신 지연 허용 (실시간성 불필요)                   │
│  ✅ 일부 기능 장애 허용 (핵심 조회 기능만 유지)               │
└─────────────────────────────────────────────────────────────┘
```

### 5.1 주요 위험 요소

| 위험 | 영향도 | 발생 가능성 | 대응 방안 |
|------|--------|------------|-----------|
| 크롤러 고장 | **중간** | 높음 | 장애 알림 → 외부 전문가 (1-2일 내 복구) |
| 보안 취약점 | 중간 | 낮음 | 내부망 접근 + 연 1회 점검 |
| AI API 비용 증가 | 낮음 | 낮음 | 사용량 모니터링 (내부 사용이라 제한적) |
| Vercel 장애 | 중간 | 낮음 | 대기 (Vercel 복구 시 자동 복구) |
| 담당자 이탈 | 중간 | 중간 | 문서화 + 엑셀 업로드 단순화로 진입장벽 낮춤 |

> **참고**: 내부 서비스 특성상 대부분의 장애는 "대기 후 복구" 전략 적용 가능

### 5.2 비상 대응 체계 (단순화)

```
Level 1 (자체 해결): 즉시
  - 엑셀 업로드 오류 → 파일 형식 확인 후 재시도
  - 페이지 오류 → 새로고침 또는 재로그인

Level 2 (대기): 1-2일
  - 뉴스 수집 중단 → 크롤러 장애 알림 확인
  - 외부 전문가 연락 (급하지 않으면 대기)

Level 3 (긴급 대응): 필요시
  - 전체 서비스 접근 불가 → Vercel 상태 확인
  - 데이터 손실 의심 → 외부 전문가 긴급 연락
```
